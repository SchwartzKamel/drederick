## 5. Idempotency & Safety ✅

### Workflow Idempotency
The workflow is **not idempotent** by design, which is correct for releases:
- GitHub Actions prevents duplicate tag-triggered workflows (same tag = same release)
- `workflow_dispatch` with explicit `version` input allows manual re-runs for specific tags
- Concurrency group `release-${github.ref}` ensures only one release per tag runs simultaneously

### Cleanup Strategy ✅
- Temporary directories in build jobs created with `mktemp` and cleaned properly
- Artifacts cleaned up automatically (retention-days: 7 on upload-artifact)
- No persistent state pollution across runs
- install.sh also cleans temp files with trap 'rm -rf "$TMPDIR"' EXIT
